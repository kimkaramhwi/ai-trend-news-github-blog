## 한 줄로 말하면

SQLite(경량 내장형 데이터베이스)를 단순한 데이터 저장소가 아니라 **게임 엔진 그 자체**로 사용해, 둠(Doom)과 비슷한 1인칭 슈팅 게임을 순수 SQL만으로 만든 프로젝트입니다.

---

## 왜 중요해요

보통 게임 엔진은 C++이나 전용 프레임워크로 만들어집니다. SQLite는 데이터를 *저장*하는 용도로 쓰이죠. DOOMQL은 이 상식을 뒤집어, **움직임·충돌 판정·적 AI·전투·화면의 모든 RGB 픽셀**까지 SQL이 직접 처리하게 만들었습니다.

이 프로젝트가 흥미로운 이유는 두 가지입니다.

1. **SQL의 가능성 실험** — "SQL로 과연 어디까지 할 수 있을까?"라는 엉뚱한 질문에 실제로 답을 내놓았습니다.
2. **AI 활용 사례** — Peter Gostev가 GPT-5.6 Sol을 활용해 만들었고, 시연 과정에서도 Claude(Fable 5)를 써서 부가 기능을 빠르게 추가했습니다.

---

## 자세히

### SQLite가 게임 엔진이 되는 원리

DOOMQL의 핵심은 **재귀 CTE(Recursive CTE, 자기 자신을 참조하며 반복 계산하는 SQL 구문)**입니다. 이 거대한 SQL 쿼리 하나가 레이 트레이서(Ray Tracer, 빛의 경로를 계산해 3D 화면을 그리는 기술)를 구현합니다. 즉, 화면에 보이는 벽·적·색상 픽셀 하나하나가 모두 SQL 연산의 결과물입니다.

### 실행 방법

- **Python 터미널 스크립트** 형태로 구현되어 있습니다.
- 게임을 실행하면 `/tmp/doomql/.doomql/doomql.sqlite` 경로에 SQLite 데이터베이스 파일이 생성됩니다.
- 이 데이터베이스를 **Datasette(SQLite DB를 웹에서 탐색할 수 있게 해주는 도구)**로 열어볼 수 있습니다.

### Datasette로 화면 들여다보기

Simon Willison은 `--with datasette-apps` 옵션으로 **Datasette Apps 플러그인**을 설치했습니다. 이 플러그인은 Datasette 안에서 SQL 쿼리를 직접 실행하는 커스텀 HTML+JavaScript 앱을 만들 수 있게 해줍니다.

그는 Claude(Fable 5)에게 다음 두 가지를 요청했습니다.

1. `frame_pixels` 뷰의 `x, y, r, g, b` 컬럼을 읽어 현재 화면 상태를 1초마다 갱신해서 보여주는 앱 만들기
2. 미니맵(minimap) 추가하기

두 요청 모두 간단한 프롬프트 한 줄로 바로 동작하는 앱이 만들어졌습니다. 터미널에서 게임을 플레이하면서, Datasette 앱에서 실시간으로 화면 상태와 미니맵을 함께 볼 수 있게 된 것입니다.

### 핵심 정리

| 구성 요소 | 역할 |
|---|---|
| SQLite + 재귀 CTE | 게임 로직 및 레이 트레이싱 전체 처리 |
| Python 스크립트 | 실행 진입점 (터미널 게임 화면 출력) |
| Datasette + Apps 플러그인 | DB 탐색 및 실시간 화면 시각화 |
| GPT-5.6 Sol | 프로젝트 제작에 활용된 AI 모델 |
| Claude (Fable 5) | Datasette 앱 코드 생성에 활용 |

> "SQLite를 단순한 저장소가 아니라 게임 엔진으로 쓴다"는 **의도적으로 무모한 질문**에서 시작된 이 프로젝트는, AI 코딩 도구와 결합했을 때 얼마나 빠르게 아이디어를 현실로 옮길 수 있는지도 잘 보여주는 사례입니다.
