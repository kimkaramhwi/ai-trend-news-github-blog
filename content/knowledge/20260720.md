## 한 줄로 말하면

Cline SDK가 v0.0.65로 업데이트되면서 **설치 파일 크기가 확 줄고**, 팀 단위 AI 에이전트 실행의 안정성이 높아졌습니다.

## 왜 중요해요

Cline은 AI가 코드를 직접 작성·실행할 수 있도록 돕는 에이전트(agent, 자동으로 작업을 수행하는 AI 프로그램) 도구입니다. SDK(Software Development Kit, 소프트웨어 개발 도구 모음)는 개발자들이 Cline의 기능을 자신의 프로젝트에 가져다 쓸 수 있게 해주는 패키지입니다.

이번 업데이트의 핵심은 크게 두 가지입니다.

1. **설치 용량 감소** — 무거운 라이브러리를 항상 함께 설치하지 않아도 되도록 바꿔, 프로젝트를 처음 세팅할 때 드는 시간과 용량을 아낄 수 있습니다.
2. **팀 실행 신뢰도 향상** — 여러 AI 에이전트가 협력하는 팀 실행 환경에서 발생하던 오류 처리 문제를 바로잡았습니다.

## 자세히

### 📦 선택적 의존성으로 설치 용량 절감
기존에는 Claude Code와 Codex 프로바이더(provider, 특정 AI 모델을 연결해 주는 모듈)가 SDK를 설치할 때 무조건 함께 설치됐습니다. 이번 버전부터는 **필요할 때만 불러오는 선택적 피어 의존성(optional peer dependency)**으로 바뀌어, 실제로 사용하는 경우에만 해당 라이브러리가 설치됩니다. 덕분에 설치 크기가 크게 줄었습니다.

### 🤖 Kimi K3 모델 추가
ClinePass(Cline의 자체 구독 서비스)의 기본 모델 목록에 **Kimi K3**가 새로 추가되었습니다.

### 🔄 만료된 인증 자동 갱신 후 재시도
OAuth(OAuth, 로그인 정보를 안전하게 위임하는 인증 방식) 토큰이 만료되어 실행이 실패한 경우, 이제 자격증명을 자동으로 갱신한 뒤 **한 번 더 재시도**합니다. 사용자가 직접 로그인을 다시 하지 않아도 됩니다.

### 👥 팀 실행 관련 개선
- **spawn 도구 제한**: 팀 실행 환경에서 teammate(팀원 역할의 하위 에이전트)에게 `spawn` 도구(새 에이전트를 생성하는 도구)가 더 이상 노출되지 않습니다. 의도치 않은 에이전트 생성을 방지하는 안전장치입니다.
- **오류 상태 정확히 표시**: 팀원 에이전트 실행 중 오류가 발생하면 이전에는 '완료'로 잘못 표시되던 것이, 이제 정확히 **'실패'로 보고**됩니다.

### 🪟 Windows 셸 호환성 수정
셸 명령어(shell command, 운영체제에 직접 내리는 명령) 파싱(parsing, 명령어를 분석·해석하는 과정) 로직을 개선해 Windows 환경에서 발생하던 셸 불일치 문제를 수정했습니다.

### 🎨 새 UI 컴포넌트 패키지
`@cline/ui`라는 새 패키지가 추가되었습니다. 에이전트 채팅 UI 컴포넌트를 Storybook(컴포넌트를 독립적으로 개발·확인하는 도구)으로 확인하고 npm(Node.js 패키지 관리자)으로 배포할 수 있게 되었습니다.
