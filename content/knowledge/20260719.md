## 한 줄로 말하면

AirLLM은 단 4GB VRAM(비디오 메모리)짜리 GPU 한 장으로 700억(70B) 개 파라미터짜리 거대 언어 모델(LLM, Large Language Model)을 돌릴 수 있게 해주는 오픈소스 라이브러리입니다. 양자화(Quantization, 숫자 정밀도를 낮춰 용량을 줄이는 기법)·증류(Distillation, 큰 모델의 지식을 작은 모델에 옮기는 기법)·가지치기(Pruning, 모델 일부를 제거해 경량화하는 기법) 없이 말이죠.

## 왜 중요해요

보통 70B 모델을 실행하려면 수십 GB의 VRAM이 필요해 고가의 서버급 GPU가 필수였습니다. AirLLM은 이 장벽을 획기적으로 낮춰줍니다.

- **일반 소비자 GPU로 초대형 모델 실행 가능**: 4GB GPU로 70B 모델, 8GB로 Llama 3.1 405B, 약 12GB로 DeepSeek-V3(671B)까지 실행할 수 있다고 원문은 밝히고 있습니다.
- **모델 구조를 건드리지 않음**: 모델 자체를 변형하거나 정확도를 희생하는 기존 경량화 방식과 달리, 원본 모델을 그대로 사용합니다.
- **진입 장벽이 낮음**: 허깅페이스(Hugging Face) 저장소 ID 하나만 넘겨주면 대부분의 인기 오픈 LLM을 바로 실행할 수 있습니다.

## 자세히

### 핵심 원리: 한 번에 레이어 하나만

AirLLM의 비결은 간단합니다. **GPU에 모델 전체를 올리지 않고, 레이어(Layer, 신경망의 층) 하나씩만 올려서 계산**합니다. 이렇게 하면 필요한 VRAM이 모델의 전체 크기가 아니라 레이어 하나의 크기에만 의존하게 됩니다. 671B짜리 거대 모델도 레이어 단위로 쪼개면 일반 GPU에서 처리할 수 있는 이유가 바로 이것입니다.

첫 실행 시에는 원본 모델을 레이어별로 분리해 디스크에 저장하는 과정이 진행됩니다. 따라서 원문에서도 허깅페이스 캐시 디렉토리에 충분한 디스크 공간이 있어야 한다고 강조합니다.

### 속도를 높이는 모델 압축 옵션

AirLLM은 블록 단위 양자화(Block-wise Quantization) 기반의 선택적 압축 기능도 제공합니다. 이를 통해 추론 속도를 최대 3배까지 높일 수 있다고 원문은 설명합니다.

일반적인 양자화는 가중치(Weights)와 활성화값(Activations) 모두를 양자화해야 해서 정확도 유지가 어렵습니다. 반면 AirLLM은 **디스크에서 불러오는 속도가 병목**이기 때문에, 가중치만 양자화해도 충분합니다. 덕분에 정확도 손실을 최소화할 수 있습니다.

압축은 모델 초기화 시 `compression='4bit'` 또는 `compression='8bit'` 인자를 전달하는 것만으로 활성화됩니다.

### 지원 모델과 설정 옵션

AutoModel 하나로 사실상 모든 주요 오픈 LLM을 지원합니다. 원문이 언급하는 지원 모델 계열은 다음과 같습니다:

- **Llama** 2 / 3 / 3.1 / 3.3 / 4
- **Qwen** 1 / 2 / 2.5 / 3 (MoE, FP8 포함)
- **DeepSeek** V2 / V3 / R1
- **Mistral & Mixtral**, **Phi**, **Gemma**, **ChatGLM**, **Baichuan**, **InternLM**, **Yi**

초기화 시 사용할 수 있는 주요 설정 옵션도 원문에 명시되어 있습니다:

| 옵션 | 설명 |
|---|---|
| `compression` | `'4bit'` 또는 `'8bit'` 압축, 기본값은 None |
| `prefetching` | 모델 로딩과 연산을 겹쳐서 속도 개선 (기본 활성화) |
| `hf_token` | 허깅페이스 비공개(gated) 모델 접근용 토큰 |
| `delete_original` | 원본 모델 삭제로 디스크 공간 절약 |
| `layer_shards_saving_path` | 분리된 레이어 모델을 저장할 경로 지정 |

### MacOS 지원

Apple Silicon Mac에서도 동일하게 사용할 수 있으며, mlx와 torch 설치가 필요합니다. Intel Mac은 지원되지 않습니다.
